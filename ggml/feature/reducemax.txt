ReduceMax Feature Notes

Overview
- Added native GGML operator support for ReduceMax.
- Current ggml reduction primitive is row-wise reduction on dimension ne[0].
- Runtime supports ONNX-style axis by permuting target axis to ggml dim-0 before reduce, then optionally permuting back.

1) ggml native op + API
File: ggml/include/ggml.h

- API declaration:
  - ggml_reduce_max(ctx, a)

API contract
- input type: F16/F32 only
- output type: same as input
- reduction dimension: ggml dim-0 (row-wise)
- output shape: [1, ne1, ne2, ne3]

ggml_reduce_max() behavior
- validates type is F16/F32
- allocates output with ne[0] forced to 1, other dims unchanged
- sets op = GGML_OP_REDUCE_MAX
- stores src[0] = input tensor

Kernel details
- one-thread compute path (ith != 0 returns)
- F32: uses ggml_vec_max_f32 over ne00
- F16: loops over ne00, converts fp16->fp32 for max accumulation, writes fp16 result
- requires output ne0 == 1 and remaining dims equal input

5) Runtime ONNX ReduceMax mapping (important)
File: runtime/src/model_runtime.cpp
- ReduceMax node handling starts
  - around line 1438
- supports input dtype F16/F32
- supports axis from attributes or second input tensor
- currently supports single-axis reduction only
- maps ONNX axis to ggml axis using existing runtime mapping
- if axis != ggml dim-0:
  - permute target axis to front
  - call ggml_reduce_max
  - permute result back
- keepdims handling:
  - keepdims=1: keep reduced axis as size-1 dim
  - keepdims=0: squeeze reduced axis via reshape_1d/2d/3d

Notes and limitations
- ggml primitive itself has no axis parameter; axis support is implemented in runtime by permute strategy.
- runtime currently allows only one axis for ReduceMax.
- behavior for axis=-1 depends on existing axis mapping helper used by your runtime.
